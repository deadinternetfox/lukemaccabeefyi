ğŸ—ï¸ How It Works

1. Document Vectorization: On server startup, markdown documents describing my services and experience are converted into vector embeddings using OpenAI's embedding API.

2. Vector Storage: These embeddings are stored in Pinecone, enabling fast semantic search.

3. Query Processing: When a user asks a question, it's converted to a vector and matched against stored documents.

4. Context Injection: Relevant document snippets are injected into the AI prompt as context.

5. Response Generation: The AI generates a response using both the context and conversation history.

ğŸ§— Challenges & Solutions

Challenge: Response Accuracy

Generic AI responses didn't reflect my actual services and experience accurately.

Solution: Implemented RAG with Pinecone to ground responses in my actual documentation, dramatically improving accuracy.

Challenge: Cost Management

GPT-5 API calls can be expensive with high traffic.

Solution: Added NovelAI as an alternative engine and implemented session limits to manage costs.

Challenge: Deployment

Needed secure, reliable hosting that works alongside existing projects.

Solution: Used Cloudflare Tunnels for secure access and PM2 for process management on the same server as FrogMaps.
